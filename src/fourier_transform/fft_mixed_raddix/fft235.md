# Radix-2, 3, 4, 5 and 8 FFT Routine

## Algorithm

The main mixed-radix FFT driver for the FFTE library. Decomposes a transform of length `n = 2^a · 3^b · 5^c` into a sequence of small-radix FFT stages (radix-8, 5, 4, 3, 2), applying precomputed twiddle factors between stages. Also provides routines for initializing twiddle factor tables.

### FFT235: Mixed-Radix FFT Driver

```
function FFT235(a, b, w, n, ip):
    // ip = [a, b, c] where n = 2^a · 3^b · 5^c
    // Decompose power-of-2 part: a = 3·kp8 + 2·kp4 + (0 or 1)
    kp4 = 2 - (a + 2) mod 3
    kp8 = (a - 2·kp4) / 3

    key = +1          // tracks which buffer holds current data
    l = n, m = 1      // l = remaining length, m = stride

    // Apply radix-8 stages
    for each of kp8 stages:
        l = l / 8
        FFT8(src, dst, w[j:], m, l)    // src/dst depend on key
        key = -key
        m = m * 8

    // Apply radix-5 stages
    for each of ip[2] stages:
        l = l / 5
        FFT5(src, dst, w[j:], m, l)
        key = -key
        m = m * 5

    // Apply radix-4 stages
    for each of kp4 stages:
        l = l / 4
        FFT4(src, dst, w[j:], m, l)
        key = -key
        m = m * 4

    // Apply radix-3 stages
    for each of ip[1] stages:
        l = l / 3
        FFT3(src, dst, w[j:], m, l)
        key = -key
        m = m * 3

    // Final radix-2 stage (if a == 1)
    if ip[0] == 1:
        FFT2(src, a, m)
```

### SETTBL: Twiddle Factor Table Setup

```
function SETTBL(w, n):
    // Compute twiddle factors for each stage
    for each stage with radix r and group size l:
        for j = 0 to l-1:
            for i = 1 to r-1:
                w[i-1][j] = e^(-2πi · i · j / (r · l))
```

### SETTBL2: 2-D Twiddle Factor Table

```
function SETTBL2(w, nx, ny):
    for j = 0 to ny-1:
        for i = 0 to nx-1:
            w[i][j] = e^(-2πi · i · j / (nx · ny))
```

### Notes

- The power-of-2 exponent `a` is decomposed as `a = 3·kp8 + 2·kp4 + (0 or 1)` to maximize radix-8 stages, which are the most efficient per butterfly.
- The `key` variable tracks which of the two buffers (`a` or `b`) currently holds the data, avoiding unnecessary copies.
- When `l == 1` (last stage for a given radix), the output is written in-place to avoid an extra copy.
- Each `FFTr` kernel has two variants: `FFTrA` (stride `m == 1`, optimized) and `FFTrB` (general stride `m > 1`).
- The twiddle factor layout is stage-sequential: all factors for stage 1, then stage 2, etc.
